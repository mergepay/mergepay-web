# Drips Wave — `mergepay-web`

The Mergepay Drips Wave is run from the **backend** repo so that contributions never
touch the finished frontend. The full maintainer guide and the bounty-ready issue
queue live there:

➡️ **[mergepay/mergepay-api → DRIPS_WAVE.md](https://github.com/mergepay/mergepay-api/blob/main/DRIPS_WAVE.md)**

This `mergepay-web` repo still carries everything it needs to be **independently
claimable** on Drips:

- [x] Hosted under the `mergepay` GitHub Organization (not a personal account).
- [x] Open-source [`LICENSE`](LICENSE) (MIT).
- [x] [`FUNDING.json`](FUNDING.json) at the repo root on the default branch.
- [x] README with the Stellar ecosystem utility statement + a <5-minute setup guide.
- [x] [`CONTRIBUTING.md`](CONTRIBUTING.md) with PR standards, public-communication rule, and the claim flow.

## Human steps only you can do

- [ ] Confirm the repo is **Public**.
- [ ] Set your Ethereum address in [`FUNDING.json`](FUNDING.json) (replace the `0x0000…0000` placeholder) and commit to `main`.
- [ ] Claim the project in the **Drips App** (connect wallet → Projects → Claim — it verifies `FUNDING.json`).
- [ ] Install the **Drips Wave GitHub App** on the `mergepay` org.
- [ ] Complete **KYC / identity verification** on Drips before distributing rewards.

> **Where to apply with issues:** apply the Wave with `mergepay-api`, which has the
> scoped backend issue queue. Only open `drips-wave` issues here if you later want
> frontend contributions.
